IDE support for Multiversion, a compile-time mixin system for Minecraft mods that lets you maintain a single codebase across multiple game versions.

Instead of maintaining separate forks per version, you write version-specific patches using familiar Mixin-style annotations. The Gradle plugin merges your version layers into a unified source tree at build time with zero runtime overhead. Whenever part of a class breaks in a more recent Minecraft version, you essentially only need to write a mixin-like patch that fixes the relevant change, vastly reducing code duplication.

IDE Features

  • Live patchedSrc updates: saves, structural edits (adding/removing members), new file creation, and file deletion all trigger an immediate re-merge of patchedSrc with cascading to downstream versions. Run generateAllPatchedSrc manually only for a full codebase refresh.
  • Go to Declaration navigates to the correct original source file and line.
  • Find Usages searches patchedSrc and maps results back to real source files.
  • Suppresses false errors caused by multiple source sets and compile-time-only annotations.
  • Refactoring support: renames and moves propagate across all version modules.

See the GitHub repository for setup instructions, Gradle plugin documentation, and the annotation reference.
