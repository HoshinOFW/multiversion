IDE support for Multiversion, a compile-time mixin system for Minecraft mods that lets you maintain a single codebase across multiple game versions.

Instead of maintaining separate forks per version, you write version-specific patches using familiar Mixin-style annotations. The Gradle plugin merges your version layers into a unified source tree at build time with zero runtime overhead. Whenever part of a class breaks in a more recent Minecraft version, you essentially only need to write a mixin-like patch that fixes the relevant change, vastly reducing code duplication.

IDE Features

  • Live patchedSrc updates: saves, structural edits, file creation, and file deletion trigger an immediate re-merge with cascading to downstream versions.
  • Version navigation: Alt+W / Alt+S to jump upstream/downstream. Context-sensitive: navigates the member at cursor, or the class.
  • All-versions popup: Alt+Shift+V shows every version of the current member or class with status labels.
  • Gutter icons: up/down arrows on annotated members and members overridden downstream for quick navigation.
  • Error suppression: hides false duplicate class errors, uninitialized field warnings on shadow members, and other false positives.
  • Annotation validation: validates descriptor strings and flags missing annotations on members that exist in the base version.
  • Navigation: Go to Declaration and Find Usages work correctly across version layers via patchedSrc remapping.
  • Refactoring: renames and moves propagate across all version modules, including descriptor strings in annotations.

See the GitHub repository for setup instructions, Gradle plugin documentation, and the annotation reference.
