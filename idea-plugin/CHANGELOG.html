0.6.1

  • Bodyless-member navigation fix Clicking a @ModifySignature row (or any declaration without a body) in the Alt+Shift+V popup now lands on the member's real declaration in the destination version's trueSrc, instead of the class declaration line. Powered by the engine's new decl-column in the origin map. Requires running generateAllPatchedSrc once after upgrading so the new v2-format origin maps are produced; old maps are refused with a clear error.
  • Arrows + keybinds filter to real declarations Gutter arrows and Alt+Shift+W/S now navigate only between versions that carry a real declaration (@OverwriteVersion, @ModifySignature, or brand-new). @ShadowVersion-only destinations are skipped since they're references, not bodies. The arrow stops appearing when the only upstream/downstream version carries just a shadow reference.
  • Alt+Shift+V popup dims non-navigable rows @ShadowVersion-only, (absent), and (inherited) rows are rendered muted and are not clickable (previously, clicks on (absent) / (inherited) were silently swallowed). Shadow rows keep the annotation color at reduced saturation so they're still visually distinct.
  • Engine-owned origin-map reads NavigationBridge no longer parses origin-map value strings directly. All path / line / col lookups go through OriginResolver, which expands the new compact V:S:L:C format against the project's version list and per-version ClassRoutingMap.
  • Cache wired for the new resolver MergeEngineCache.resolverForModuleRoot supplies the project's ordered version list and per-version routing lookup so the resolver can turn (V, S) back into a real sibling file path.
  • Loud synthesis fallback When the IDE falls back to in-memory origin-map synthesis (unbuilt version / mid-edit file), per-file failures are now logged at WARN instead of being silently swallowed. After a full generateAllPatchedSrc, the base version has a real on-disk map and this fallback isn't used for it.
  • Move refactor warns on routing-order shifts Renaming a @ModifyClass modifier in a way that changes its alphabetical position inside a sibling group logs a warning suggesting a full rebuild, since sibling indices in _originMap.tsv may be stale until regenerated.
  • @ModifyClass navigation MergeEngineCache loads per-target routing sidecars produced by the merge engine, with engine-synthesized fallback for unbuilt versions. NavigationBridge translates the caret file's rel through the current-version routing, so navigating from a cross-name modifier (e.g. FooPatch.java) operates on the target class's identity. openMemberHit / openClassHit use origin-map values to land on the real source file regardless of name.
  • Alt+Shift+V popup Fans out one row per sibling in versions whose target is assembled from multiple @ModifyClass modifiers, each labelled with the modifier's filename and opening the specific sibling on click.
  • Patched-src refresh on modifier edits PatchedSrcUpdater is a per-module routing-aware dispatch. When the edited rel participates in a @ModifyClass sibling group in a given module, the update calls the engine's new file-level sibling-group entry (pre-merge + merge for just that group); otherwise it uses the existing single-file fast path. Downstream cascade propagates at the resolved target rel, so cross-name modifiers cascade correctly. Gradle's full-version merge path (versionUpdatePatchedSrc) is untouched and never called from the IDE listeners.
  • MoveListener @ModifyClass awareness Class rename / move delegates sidecar maintenance to the engine's new ClassRoutingMap.Sidecars.renameTarget / renameModifier helpers. The IDE never touches .routing files directly, matching how it reads / writes the origin map through OriginMap rather than raw TSV I/O.
  • @ModifyClass inspections (three) Yellow warning when a class participates in a multi-file routing group without explicit @ModifyClass (quick fix adds it). Red error on @ModifyClass(Foo.class) when the target does not exist in any earlier version's patchedSrc or when it resolves to an inner class. Red error on @ModifyClass placed on a nested class. All three mirror the merge engine's hard validation contracts.
  • MODIFY_CLASS_FQN constant added to AnnotationFqns.kt alongside the existing multiversion annotation FQNs.
  • Rename propagation across cross-name siblings MultiversionRenameProcessor and its findReferences override now enumerate every cross-name @ModifyClass sibling in each other version via the new routing-aware findCorrespondingModifierFiles helper (and process each sibling's primary top-level class, not just filename matches). Previously, renaming a member of Foo would miss @OverwriteVersion / @ShadowVersion / @ModifySignature counterparts declared inside sibling modifier files like FooExt.java — only same-rel files in other versions were scanned.
  • MoveListener TSV patch routing-aware When renaming a member inside a cross-name modifier, the origin map's member keys are stored under the target's rel (not the modifier's rel). updateTsvForRename now resolves the per-version target rel via MergeEngineCache.routingForModuleRoot before calling OriginMap.renameMember, so entries like com/.../Foo.java#bar() get updated when bar() is renamed inside a sibling modifier.
  • Known limitation Adding or removing @ModifyClass on a file (creating or dissolving a sibling group) is not picked up by the per-save incremental updater until the next full build; the routing cache is keyed on _originMap.tsv mtime which doesn't change until Gradle runs. Workaround: run generatePatchedJava after any @ModifyClass addition / removal. Edits inside an already-established group are fully incremental and do not need this.
  • BUG FIX PatchedSrc changes now properly trigger re-analysis of dependent files in open editors, including downstream versions
  • Performance Gutter icon override detection now uses origin map lookups instead of opening PSI files across all versions
  • EngineConfigCache now exposes origin maps per module using the engine's CachedOriginMap
  • Deduplicated path extraction, PSI file opening, and member matching code across the plugin
  • Navigation rebuilt on origin map Gutter arrows, Alt+Shift+W/S shortcuts, and the Alt+Shift+V popup all gate on and walk through the engine's OriginNavigation walker. Up/down arrows now appear if and only if the tracked member or class has a trueSrc version in that direction, at any distance, following @ModifySignature rename chains
  • Alt+Shift+V popup Shows every version with labels derived directly from the origin map flag set (e.g. @ShadowVersion @ModifySignature). Annotation tokens render bold in the editor's annotation colour (theme-aware). Inherited and absent entries are greyed out. Class-level entries show the trueSrc filename (future-proofed for @ModifyClass) instead of a generic label. Inherited entries are non-clickable for now
  • Base-version synthesis Navigation walks and the Alt+Shift+V popup now work on the project's base version (oldest) even though it has no generated origin map; the IDE synthesises one in memory from the trueSrc directory
  • BUG FIX Add @OverwriteVersion / Add @ShadowVersion / Replace @ShadowVersion with @OverwriteVersion / Add original annotations quick fixes now add the necessary imports if they are missing
  • BUG FIX Missing method body, or declare abstract error is now suppressed when the method matches a member in the previous version, so the Add @ShadowVersion quick fix is the only one offered on bodyless multiversion stubs
  • Deleted navigation-only helpers from VersionMemberUtil (hasAdjacentVersion, findAdjacentVersionClass, findAllVersionClasses, findAllVersionsOfMember, walkUpstream/walkDownstream, isOverriddenDownstream); IDE navigation is now a thin adapter over the engine walker
  • Some refactors

0.6.0

  • All-versions popup Alt+Shift+V shows every version of the current member or class with status labels, click to navigate.
  • Gutter icons Up/down arrows on annotated members, members overridden downstream, and versioned class declarations for quick navigation.
  • Centralized version directory scanning, member resolution, and path classification utilities across the plugin.
  • Missing explicit annotation inspection Errors when a member exists in the original class but has no @OverwriteVersion or @ShadowVersion, with a quick fix to add the appropriate one
  • @ModifySignature collision detection Errors when the new member name already exists in the original class as a different member
  • Missing original annotations inspection Warns when @OverwriteVersion or @ShadowVersion members are missing annotations present on the original member, with a quick fix to add them
  • BUG FIX @NotNull @ShadowVersion fields no longer show a spurious "must be initialized" warning
  • Config parsing now uses engine's shared EngineConfig.fromJson() instead of a local regex parser
  • Origin map caching now delegates to engine's CachedOriginMap
  • Descriptor resolution now uses engine's shared MemberDescriptor utilities (parsing, init disambiguation, param matching)
  • BUG FIX @ModifySignature with a method body but no @OverwriteVersion now correctly shows an error suggesting @OverwriteVersion
  • @ShadowVersion body warning @ShadowVersion methods or fields with a body now show a warning with quick fixes to replace with @OverwriteVersion or remove the body
  • BUG FIX "Missing explicit annotation" inspection now correctly suggests @OverwriteVersion for fields with initializers (previously always suggested @ShadowVersion for fields)
  • Some refactoring for future maintenance
