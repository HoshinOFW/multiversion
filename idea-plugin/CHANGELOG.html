0.6.1

  • BUG FIX PatchedSrc changes now properly trigger re-analysis of dependent files in open editors, including downstream versions
  • Performance Gutter icon override detection now uses origin map lookups instead of opening PSI files across all versions
  • EngineConfigCache now exposes origin maps per module using the engine's CachedOriginMap
  • Deduplicated path extraction, PSI file opening, and member matching code across the plugin
  • Navigation rebuilt on origin map Gutter arrows, Alt+Shift+W/S shortcuts, and the Alt+Shift+V popup all gate on and walk through the engine's OriginNavigation walker. Up/down arrows now appear if and only if the tracked member or class has a trueSrc version in that direction, at any distance, following @ModifySignature rename chains
  • Alt+Shift+V popup Shows every version with labels derived directly from the origin map flag set (e.g. @ShadowVersion @ModifySignature). Annotation tokens render bold in the editor's annotation colour (theme-aware). Inherited and absent entries are greyed out. Class-level entries show the trueSrc filename (future-proofed for @ModifyClass) instead of a generic label. Inherited entries are non-clickable for now
  • Base-version synthesis Navigation walks and the Alt+Shift+V popup now work on the project's base version (oldest) even though it has no generated origin map; the IDE synthesises one in memory from the trueSrc directory
  • BUG FIX Add @OverwriteVersion / Add @ShadowVersion / Replace @ShadowVersion with @OverwriteVersion / Add original annotations quick fixes now add the necessary imports if they are missing
  • BUG FIX Missing method body, or declare abstract error is now suppressed when the method matches a member in the previous version, so the Add @ShadowVersion quick fix is the only one offered on bodyless multiversion stubs
  • Deleted navigation-only helpers from VersionMemberUtil (hasAdjacentVersion, findAdjacentVersionClass, findAllVersionClasses, findAllVersionsOfMember, walkUpstream/walkDownstream, isOverriddenDownstream); IDE navigation is now a thin adapter over the engine walker
  • Some refactors

0.6.0

  • Version navigation Alt+Shift+W/S to jump upstream/downstream versions; context-sensitive (navigates the member at cursor, or the class). Replaces the current tab in-place.
  • All-versions popup Alt+Shift+V shows every version of the current member or class with status labels, click to navigate.
  • Gutter icons Up/down arrows on annotated members, members overridden downstream, and versioned class declarations for quick navigation.
  • Centralized version directory scanning, member resolution, and path classification utilities across the plugin.
  • Missing explicit annotation inspection Errors when a member exists in the original class but has no @OverwriteVersion or @ShadowVersion, with a quick fix to add the appropriate one
  • @ModifySignature collision detection Errors when the new member name already exists in the original class as a different member
  • Missing original annotations inspection Warns when @OverwriteVersion or @ShadowVersion members are missing annotations present on the original member, with a quick fix to add them
  • BUG FIX @NotNull @ShadowVersion fields no longer show a spurious "must be initialized" warning
  • Config parsing now uses engine's shared EngineConfig.fromJson() instead of a local regex parser
  • Origin map caching now delegates to engine's CachedOriginMap
  • Descriptor resolution now uses engine's shared MemberDescriptor utilities (parsing, init disambiguation, param matching)
  • BUG FIX @ModifySignature with a method body but no @OverwriteVersion now correctly shows an error suggesting @OverwriteVersion
  • @ShadowVersion body warning @ShadowVersion methods or fields with a body now show a warning with quick fixes to replace with @OverwriteVersion or remove the body
  • BUG FIX "Missing explicit annotation" inspection now correctly suggests @OverwriteVersion for fields with initializers (previously always suggested @ShadowVersion for fields)
  • Some refactoring for future maintenance

0.5.3

  • Auto-update on structural changes Adding or removing a method, field, or inner class now triggers a patchedSrc re-merge in the background without saving.
  • File deletion cascade Deleting a versioned source file now triggers a cascading patchedSrc recompute for the affected version and all downstream versions.
  • New file detection Creating a new .java file in a versioned source directory automatically merges it into patchedSrc.
  • Origin map consistency The origin map is now kept up to date across all update paths (save, structural change, file creation, file deletion), fixing stale navigation after incremental edits.
  • BUG FIX Multiversion annotations (@OverwriteVersion, @ShadowVersion, etc.) no longer leak into merged patchedSrc output as imports.
  • BUG FIX Edits to the base version (e.g. 1.20.1) now correctly cascade to all downstream patchedSrc (1.21.1, etc.).
  • BUG FIX Fixed SlowOperations violation when opening files in patchedSrc.
