0.5.4

  • Many many smaller bugfixes and changes, as well as refactors for future maintainability.

0.5.3

  • Auto-update on structural changes Adding or removing a method, field, or inner class now triggers a patchedSrc re-merge in the background without saving.
  • File deletion cascade Deleting a versioned source file now triggers a cascading patchedSrc recompute for the affected version and all downstream versions.
  • New file detection Creating a new .java file in a versioned source directory automatically merges it into patchedSrc.
  • Origin map consistency The origin map is now kept up to date across all update paths (save, structural change, file creation, file deletion), fixing stale navigation after incremental edits.
  • BUG FIX Multiversion annotations (@OverwriteVersion, @ShadowVersion, etc.) no longer leak into merged patchedSrc output as imports.
  • BUG FIX Edits to the base version (e.g. 1.20.1) now correctly cascade to all downstream patchedSrc (1.21.1, etc.).
  • BUG FIX Fixed SlowOperations violation when opening files in patchedSrc.

0.5.2

  • Find Usages Find Usages on a method, field, or class in a versioned source root now searches patchedSrc instead of true source, then maps each result back to its real source file and line via the origin map.

0.5.1

  • Bumped merge engine version

0.5.0

  • On-save patchedSrc update Saving a versioned source file now automatically re-merges it into patchedSrc, keeping the IDE's view of the merged class in sync without a full Gradle sync.
  • Engine config bridge The Gradle plugin now writes build/multiversion-engine-config.json per version module during generateAllPatchedSrc.

0.4.7

  • BUG FIX Ctrl+Click on init and init(params) descriptor strings inside @DeleteMethodsAndFields now navigates to the correct constructor declaration.
  • BUG FIX Renaming a constructor no longer incorrectly triggers cross-version rename propagation, which would previously attempt to rename a non-existent method.
  • BUG FIX Move refactoring now propagates to all other version modules, not just newer ones.
  • BUG FIX Move refactoring now propagates when files are moved via the project view or drag-and-drop, not only when using Refactor → Move (F6).
  • BUG FIX Opening a file in patchedSrc no longer redirects to the real source when the user is already browsing patchedSrc.

0.4.6

  • Constructors All features now support constructors. See README.md for documentation.

0.4.5

  • BUG FIX Refactors wouldn't properly propagate back due to caches not updating.

0.4.4

  • BUG FIX Fixed an IDE error on startup
  • Rename propagation renaming a method, field, or class in one version automatically renames it across all other version modules
  • @Delete* descriptor sync renaming a method or field also updates its descriptor string inside @DeleteMethodsAndFields annotations in later versions, including parameter types
  • Move propagation moving a class to a different package propagates the package change and file relocation to all later version patch files
  • Propagation toggle a "Propagate to other Minecraft versions" checkbox in the rename dialog and in Settings → Tools → Multiversion controls whether cross-version propagation is active (on by default, persisted across sessions)
  • Sentinel guard all plugin features are now disabled in projects that do not have the multiversion annotations library on the classpath, preventing interference with unrelated codebases
